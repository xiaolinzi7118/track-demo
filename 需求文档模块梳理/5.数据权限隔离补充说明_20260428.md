# 数据权限隔离补充说明（2026-04-28）

## 1. 目标

为以下模块补齐部门数据权限隔离，避免跨部门可见：

- 仪表盘
- 需求管理
- 事件管理
- 数据回检

并补充开发角色专属可见性规则。

## 2. 统一可见性规则

- 内置超级管理员：可见全部数据。
- 非开发角色用户：仅可见“需求提出人主部门”在本人数据权限范围内的需求，以及其关联事件/回检数据。
- 开发角色用户（非管理员）：仅可见 `devTeamDeptId = 当前用户主部门ID` 的需求，以及其关联事件/回检数据。

## 3. 模块行为

### 3.1 需求管理

- 列表、详情、状态变更、重提均按可见性规则校验。
- 不可见需求即使知道 `requirementId` 也无法访问与操作。

### 3.2 仪表盘

- 统计口径改为“当前用户可见需求集”。
- 趋势口径改为“当前用户可见需求集”。

### 3.3 事件管理

- 事件列表/详情/统计：在部门过滤基础上，增加“关联需求可见性”过滤。
- 关联需求下拉仅返回当前用户可见需求（且状态为排期中/开发中）。
- 新增/编辑事件时，若关联了不可见需求，后端拦截。

### 3.4 数据回检

- 原有部门过滤保留。
- 新增“可见需求关联事件编码集合”过滤，仅展示当前可见需求关联事件的回检数据。

## 4. 验收场景（核心）

- 场景A：用户A和用户B各自仅有本部门数据权限，互相不可见对方需求、事件、回检与仪表盘统计。
- 场景B：开发用户D主部门为X，仅可见 `devTeamDeptId = X` 的需求及关联事件/回检。
- 场景C：越权访问（直接传 `requirementId`、事件ID、回检ID）返回无权限。
