# 埋点管理平台 - 角色权限功能设计文档

## 一、功能概述

为埋点管理平台增加基于 RBAC（基于角色的访问控制）的权限管理系统，支持：

- **角色管理**：管理员、业务人员、开发人员三种预设角色，支持自定义角色
- **菜单权限**：基于菜单和按钮级别的细粒度权限控制
- **用户管理**：用户 CRUD + 角色分配
- **动态菜单**：根据用户角色动态渲染侧边栏菜单
- **按钮权限**：通过 `v-permission` 指令控制按钮显示/隐藏

**请求：**
```json
{
  "roleCode": "TESTER",
  "roleName": "测试人员",
  "description": "测试人员",
  "status": 1
}
```

#### POST `/api/role/update` 编辑角色
**请求：** `{ "id": 4, "roleName": "测试人员", "description": "..." }`

**请求：** `{ "id": 4 }`

**参数：** `roleId=1`

**请求：**
```json
{
  "roleId": 2,
  "menuIds": [1, 2, 3, 12, 13, 7, 8]
}
```

## 六、前端实现方案

### 6.1 技术方案

| 方面 | 实现 |
|------|------|
| 状态管理 | Pinia store 存储 token、userInfo、permissions、role、menus |
| 路由守卫 | `router.beforeEach` 校验 token，跳转登录页 |
| 动态菜单 | Layout 组件从 store.menus 渲染，不再硬编码 |
| 按钮权限 | 自定义指令 `v-permission="'track-config:add'"` |
| API 请求 | Axios 拦截器自动附加 Bearer Token |

### 6.2 权限指令使用示例

```html
<!-- 只有拥有 track-config:add 权限的用户才能看到此按钮 -->
<el-button v-permission="'track-config:add'" @click="handleAdd">新增</el-button>

<!-- 编辑和删除按钮 -->
<el-button v-permission="'track-config:edit'" @click="handleEdit(row)">编辑</el-button>
<el-button v-permission="'track-config:delete'" @click="handleDelete(row)">删除</el-button>
```

## 八、初始化数据

系统首次启动时自动初始化：

1. **菜单权限数据**：22 条记录（8 个菜单 + 14 个按钮权限）
2. **角色数据**：3 个预设角色（管理员、业务人员、开发人员）
3. **管理员用户**：admin/123456，默认关联管理员角色
4. **默认权限分配**：每个角色按设计分配对应的菜单权限
