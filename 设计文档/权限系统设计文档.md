# 权限系统设计文档

## 1. 概述

本系统实现了基于角色的访问控制（RBAC），包含角色管理、菜单权限配置、用户角色分配等功能。支持菜单级别的可见性控制和按钮级别的操作权限控制。

## 2. 技术架构

### 前端
- Vue 3.4 + Pinia + Vue Router + Element Plus
- 权限指令 `v-permission` 实现按钮级权限控制
- 动态菜单渲染，根据用户角色权限决定菜单可见性

### 后端
- Spring Boot 2.7 + Spring Data JPA + H2
- HandlerInterceptor 实现 Token 认证
- PermissionChecker 实现接口级权限校验
- 内存 ConcurrentHashMap 存储 Token（单实例部署适用）

## 4. 权限控制逻辑

### 4.1 菜单可见性（由"查看"权限控制）
- 每个页面菜单有一个 `xxx:view` 按钮权限项
- 后端 `MenuService.getMenusByUserId()` 只返回用户拥有 `xxx:view` 权限的菜单
- 如果目录下没有可见的子页面，整个目录不返回
- 前端 layout 直接渲染后端返回的菜单，无权限的菜单不显示

### 4.2 按钮可见性（由操作权限控制）
- 使用 `v-permission` 自定义指令控制按钮显示
- 示例：`<el-button v-permission="'track-config:add'">新增</el-button>`
- admin 角色自动通过所有权限检查

### 4.3 接口权限（后端校验）
- 每个接口通过 `PermissionChecker.checkPermission()` 校验
- 校验流程：解析 Token → 获取 userId → 查角色 → 查菜单权限集合 → 匹配权限标识
- admin 角色跳过所有权限检查
- 权限结果缓存 5 分钟（ConcurrentHashMap）

## 5. 认证流程

1. 用户提交用户名/密码到 `POST /api/auth/login`
2. 后端验证凭据，生成 UUID Token，存入内存 TokenStore
3. 返回 Token 给前端
4. 前端存储 Token 到 localStorage，后续请求通过 `Authorization: Bearer {token}` 携带
5. 后端 AuthInterceptor 校验 Token 有效性
6. 白名单路径：`/api/auth/login`、`/api/track-data/report`、`/api/track-data/batch-report`

## 6. API 接口定义

### 6.1 认证接口

| 方法 | 路径 | 说明 | 权限 |
|---|---|---|---|
| POST | /api/auth/login | 登录 | 无 |
| POST | /api/auth/logout | 登出 | 已认证 |
| GET | /api/auth/userinfo | 获取当前用户信息（含roles、permissions） | 已认证 |
| GET | /api/auth/menus | 获取当前用户可访问菜单树 | 已认证 |

## 7. 请求/响应示例

### 登录请求
```
POST /api/auth/login
Body: { "username": "admin", "password": "123456" }
Response: { "code": 200, "data": { "token": "a1b2c3d4...", "username": "admin", "nickname": "管理员" } }
```

### 获取用户信息
```
GET /api/auth/userinfo
Header: Authorization: Bearer a1b2c3d4...
Response: {
  "code": 200,
  "data": {
    "username": "admin",
    "nickname": "管理员",
    "roles": ["admin"],
    "roleNames": ["管理员"],
    "permissions": ["dashboard:view", "track-config:view", "track-config:add", ...]
  }
}
```

### 更新角色权限
```
POST /api/role/update-menus
Header: Authorization: Bearer a1b2c3d4...
Body: { "roleId": 2, "menuIds": [1, 2, 3, 4, 5, 6] }
Response: { "code": 200, "message": "success" }
```

## 8. 前端权限使用

### v-permission 指令
```html
<el-button v-permission="'track-config:add'" type="primary">新增配置</el-button>
<el-button v-permission="'track-config:edit'" @click="handleEdit(row)">编辑</el-button>
<el-button v-permission="'track-config:delete'" @click="handleDelete(row)">删除</el-button>
```

### 代码权限判断
```javascript
import { useUserStore } from '../store/user'
const userStore = useUserStore()
if (userStore.hasPermission('track-config:add')) {
  // 有权限
}
```
