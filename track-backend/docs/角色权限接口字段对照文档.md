# 角色权限接口字段对照文档

## 1. 文档范围

- 功能：系统管理-角色管理与权限配置
- 基础路径：
  - 角色接口：`/api/role`
  - 菜单接口：`/api/menu`

---

## 2. 统一返回结构

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

---

## 3. 权限要求

| 接口 | 权限 |
|---|---|
| `GET /api/role/list` | `system-role:view` |
| `GET /api/role/detail` | `system-role:view` |
| `POST /api/role/add` | `system-role:add` |
| `POST /api/role/update` | `system-role:edit` |
| `POST /api/role/delete` | `system-role:delete` |
| `GET /api/role/menus` | `system-role:view` |
| `POST /api/role/update-menus` | `system-role:edit` |
| `GET /api/menu/tree` | `system-role:view` |
| `GET /api/menu/user-menus` | 登录即可 |
| `GET /api/menu/user-permissions` | 登录即可 |

---

## 4. 角色接口

## 4.1 角色列表

- 方法：`GET`
- 路径：`/api/role/list`

### 响应 `data`（数组）

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 角色ID |
| `roleCode` | string | 角色编码（唯一） |
| `roleName` | string | 角色名称 |
| `description` | string \| null | 描述 |
| `status` | number | 状态：`1=启用`，`0=禁用` |
| `createTime` | datetime | 创建时间 |
| `updateTime` | datetime | 更新时间 |

---

## 4.2 角色详情

- 方法：`GET`
- 路径：`/api/role/detail`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 角色ID |

### 响应 `data`

- `Role` 对象，字段同“角色列表单行结构”。

---

## 4.3 新增角色

- 方法：`POST`
- 路径：`/api/role/add`

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `roleCode` | string | 是 | 角色编码，必须唯一 |
| `roleName` | string | 是 | 角色名称 |
| `description` | string | 否 | 描述 |
| `status` | number | 否 | 默认 `1` |

### 响应 `data`

- 新增后的 `Role` 对象。

---

## 4.4 编辑角色

- 方法：`POST`
- 路径：`/api/role/update`

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 角色ID |
| `roleName` | string | 是 | 角色名称 |
| `description` | string | 否 | 描述 |
| `status` | number | 否 | 状态 |

说明：当前实现不支持修改 `roleCode`。

### 响应 `data`

- 更新后的 `Role` 对象。

---

## 4.5 删除角色

- 方法：`POST`
- 路径：`/api/role/delete`

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 角色ID |

### 响应 `data`

- `null`

---

## 4.6 查询角色菜单权限

- 方法：`GET`
- 路径：`/api/role/menus`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 角色ID |

### 响应 `data`（数组）

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 菜单ID |
| `menuCode` | string | 菜单业务标识（如 `system-role:view`） |

说明：返回当前角色已勾选的菜单集合，兼容“按 `id` 关联、按 `menuCode` 展示/识别”的前端实现。

---

## 4.7 更新角色菜单权限

- 方法：`POST`
- 路径：`/api/role/update-menus`

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `roleId` | number | 是 | 角色ID |
| `menuIds` | number[] \| string[] | 是 | 勾选菜单ID集合（支持雪花ID字符串），按此全量覆盖 |

### 响应 `data`

- `null`

---

## 5. 菜单接口

## 5.1 菜单树（角色授权用）

- 方法：`GET`
- 路径：`/api/menu/tree`

### 响应 `data`（数组，树结构）

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 菜单ID |
| `parentId` | number | 父ID |
| `menuName` | string | 菜单名称 |
| `menuCode` | string | 菜单编码 |
| `path` | string \| null | 路由 |
| `icon` | string \| null | 图标 |
| `sortOrder` | number | 排序 |
| `menuType` | number | `1=目录`，`2=页面`，`3=按钮` |
| `perms` | string \| null | 权限标识 |
| `status` | number | 状态 |
| `children` | Menu[] | 子节点 |

说明：该接口返回“全量授权树”，但会过滤 `system-dict-param` 页面。

---

## 5.2 当前用户菜单

- 方法：`GET`
- 路径：`/api/menu/user-menus`

### 响应 `data`

- `Menu[]` 树，按当前用户角色与权限动态裁剪。

---

## 5.3 当前用户权限点

- 方法：`GET`
- 路径：`/api/menu/user-permissions`

### 响应 `data`

- `string[]`：权限标识列表，如：
  - `system-user:view`
  - `system-role:edit`
  - `track-config:add`

---

## 6. 业务规则说明

1. `admin` 角色（`roleCode=admin`）不可删除。  
2. 角色下存在用户时不可删除。  
3. `update-menus` 为覆盖式更新：先删后插。  
4. 内置超级管理员账号拥有全权限，不依赖角色菜单关系。
