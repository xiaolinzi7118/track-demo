# 事件管理接口字段对照文档

## 1. 文档范围

- 功能：埋点管理-事件管理
- 基础路径：`/api/event-manage`

---

## 2. 统一返回结构

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

---

## 3. 权限要求

| 接口 | 权限 |
|---|---|
| `GET /api/event-manage/list` | `event-manage:view` |
| `GET /api/event-manage/detail` | `event-manage:view` |
| `GET /api/event-manage/requirement-options` | `event-manage:view` |
| `POST /api/event-manage/add` | `event-manage:add` |
| `POST /api/event-manage/update` | `event-manage:edit` |
| `POST /api/event-manage/delete` | `event-manage:delete` |
| `GET /api/event-manage/statistics` | `event-manage:view` |
| `GET /api/event-manage/all` | SDK 拉取配置（免登录） |

---

## 4. 事件数据结构（EventConfig）

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 主键 |
| `eventName` | string | 事件名称 |
| `eventCode` | string | 事件编码 |
| `eventType` | string | `page_view` / `click` |
| `requirementId` | string | 关联需求ID |
| `requirementTitle` | string \| null | 关联需求标题（列表展示） |
| `requirementStatus` | string \| null | 关联需求状态码 |
| `requirementStatusLabel` | string \| null | 关联需求状态中文 |
| `description` | string \| null | 事件描述 |
| `urlPattern` | string \| null | 生效页面URL正则 |
| `params` | string(JSON) | 属性引用配置数组 |
| `status` | number | `1=启用`，`0=禁用` |
| `createTime` | datetime | 创建时间 |
| `updateTime` | datetime | 更新时间 |

---

## 5. params（属性引用配置）结构

```json
[
  {
    "attributeId": "ATTR2026042700000001",
    "attributeType": "custom",
    "attributeName": "商品ID",
    "attributeField": "productId",
    "description": "详情页展示商品ID",
    "sourceType": "api_data",
    "sourceValue": "data.productId",
    "interfaceId": 1,
    "interfacePath": "/api/product/detail",
    "defaultValue": ""
  }
]
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `attributeId` | string | 对应 `track_attribute.attribute_id` |
| `attributeType` | string | `user/system/custom` |
| `attributeName` | string | 属性名称（冗余） |
| `attributeField` | string | 上报字段key |
| `description` | string \| null | 当前事件中该属性描述 |
| `sourceType` | string \| null | `custom` 时有效 |
| `sourceValue` | string \| null | `custom` 时有效 |
| `interfaceId` | number \| null | `sourceType=api_data` 时有效 |
| `interfacePath` | string \| null | 后端根据 `interfaceId` 填充 |
| `defaultValue` | string \| null | `custom` 时有效 |

---

## 6. 接口明细

## 6.1 事件列表

- 方法：`GET`
- 路径：`/api/event-manage/list`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `keyword` | string | 否 | 按 `eventName/eventCode` 模糊搜索 |
| `eventType` | string | 否 | `page_view/click` |
| `pageNum` | number | 否 | 默认 `1` |
| `pageSize` | number | 否 | 默认 `10` |

响应 `data`：分页结构，内容为 `EventConfig[]`。

---

## 6.2 需求下拉

- 方法：`GET`
- 路径：`/api/event-manage/requirement-options`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `keyword` | string | 否 | 按需求标题模糊搜索 |

响应 `data`：

```json
[
  {
    "requirementId": "uuid-string",
    "title": "需求标题",
    "status": "SCHEDULING",
    "statusLabel": "排期中"
  }
]
```

说明：仅返回状态为 `SCHEDULING/DEVELOPING` 的需求。

---

## 6.3 新增事件

- 方法：`POST`
- 路径：`/api/event-manage/add`

请求体字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `eventName` | string | 是 | 事件名称 |
| `eventCode` | string | 是 | 事件编码 |
| `eventType` | string | 是 | `page_view/click` |
| `status` | number | 是 | `1=启用`，`0=禁用` |
| `requirementId` | string | 是 | 关联需求ID |
| `description` | string | 否 | 事件描述 |
| `urlPattern` | string | 否 | 生效页面URL正则 |
| `params` | string(JSON) | 否 | 属性引用配置数组 |

响应 `data`：新增后的 `EventConfig`。

---

## 6.4 更新事件

- 方法：`POST`
- 路径：`/api/event-manage/update`

请求体字段：同新增事件，另需 `id`。

---

## 6.5 删除事件

- 方法：`POST`
- 路径：`/api/event-manage/delete`

请求体：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 事件主键ID |

响应 `data`：`null`

---

## 6.6 SDK 全量事件配置

- 方法：`GET`
- 路径：`/api/event-manage/all`

响应 `data`：`EventConfig[]`（仅 `status=1`，包含完整 `params`）。

---

## 7. 业务规则说明

1. 创建/编辑事件必须关联状态为“排期中/开发中”的需求。  
2. 事件编码 + 生效页面URL（`urlPattern`）在启用数据内唯一。  
3. `page_view` 事件不允许使用 `sourceType=node_content/api_data` 的个性化属性来源。  
4. 个性化属性来源配置按“事件维度”保存于 `track_config.params`，不回写 `track_attribute`。
