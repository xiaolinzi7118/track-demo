# 事件管理接口字段对照文档

## 1. 文档范围

- 模块：事件管理
- 基础路径：`/api/event-manage`
- 后端实现：`EventManageController`、`TrackConfigService`
- 关联表：`track_config`、`track_requirement`、`track_attribute`、`track_file_asset`

## 2. 统一返回

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

## 3. 权限要求

| 接口 | 权限 |
|---|---|
| `GET /api/event-manage/list` | `event-manage:view` |
| `GET /api/event-manage/detail` | `event-manage:view` |
| `GET /api/event-manage/requirement-options` | `event-manage:view` |
| `POST /api/event-manage/add` | `event-manage:add` |
| `POST /api/event-manage/update` | `event-manage:edit` |
| `POST /api/event-manage/delete` | `event-manage:delete` |
| `GET /api/event-manage/statistics` | `event-manage:view` |
| `GET /api/event-manage/all` | SDK拉取配置（免登录） |

## 4. EventConfig 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 主键 |
| `eventName` | string | 事件名称 |
| `eventCode` | string | 事件编码 |
| `eventType` | string | `page_view` / `click` |
| `requirementId` | string | 关联需求ID |
| `requirementTitle` | string \| null | 关联需求标题 |
| `requirementStatus` | string \| null | 关联需求状态 |
| `requirementStatusLabel` | string \| null | 关联需求状态中文 |
| `description` | string \| null | 事件描述 |
| `pageScreenshotFileId` | string \| null | 页面截图文件ID |
| `urlPattern` | string \| null | 生效页面URL正则 |
| `params` | string(JSON) | 关联属性配置 |
| `status` | number | `1=启用`，`0=禁用` |
| `createTime` | datetime | 创建时间 |
| `updateTime` | datetime | 更新时间 |

## 5. 接口明细

## 5.1 事件列表

- 方法：`GET`
- 路径：`/api/event-manage/list`
- 参数：`keyword`、`eventType`、`pageNum`、`pageSize`

## 5.2 需求下拉

- 方法：`GET`
- 路径：`/api/event-manage/requirement-options`
- 参数：`keyword`
- 说明：仅返回 `SCHEDULING/DEVELOPING` 状态需求。

## 5.3 新增事件

- 方法：`POST`
- 路径：`/api/event-manage/add`

请求体字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `eventName` | string | 是 | 事件名称 |
| `eventCode` | string | 是 | 事件编码 |
| `eventType` | string | 是 | `page_view/click` |
| `status` | number | 是 | 0/1 |
| `requirementId` | string | 是 | 关联需求 |
| `description` | string | 否 | 描述 |
| `pageScreenshotFileId` | string | 否 | 页面截图文件ID |
| `urlPattern` | string | 否 | URL正则 |
| `params` | string(JSON) | 否 | 属性引用数组 |

后端校验：

- `pageScreenshotFileId` 非空时必须存在于 `track_file_asset.file_id`。

## 5.4 更新事件

- 方法：`POST`
- 路径：`/api/event-manage/update`
- 请求体：同新增，并额外携带 `id`。

## 5.5 删除事件

- 方法：`POST`
- 路径：`/api/event-manage/delete`
- 请求体：`{ "id": number }`

## 5.6 SDK 全量配置

- 方法：`GET`
- 路径：`/api/event-manage/all`
- 说明：返回全部启用事件及完整 `params`。

## 6. 业务规则补充

1. 创建/编辑事件时，必须关联状态为“排期中/开发中”的需求。
2. `eventCode + urlPattern` 在启用数据中必须唯一。
3. `page_view` 事件不允许使用 `node_content/api_data` 的个性化属性来源。
4. 页面截图字段非必填，图片上传与预览走统一文件接口。

## 7. 依赖接口（图片）

- 上传图片：`POST /api/file/upload-image`（multipart/form-data）
- 预览图片：`GET /api/file/preview/{fileId}`（鉴权后返回二进制）
