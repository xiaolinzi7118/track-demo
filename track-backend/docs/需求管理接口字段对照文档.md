# 需求管理接口字段对照文档

## 1. 文档范围

- 模块：需求管理
- 基础路径：`/api/requirement`
- 后端实现：`RequirementController`、`RequirementService`
- 关联表：`track_requirement`、`track_log`、`track_file_asset`

## 2. 状态与动作

### 2.1 需求状态 `status`

- `PENDING_REVIEW` 待审核
- `SCHEDULING` 排期中
- `DEVELOPING` 开发中
- `TESTING` 测试中
- `ONLINE` 已上线
- `OFFLINE` 已下线
- `REJECTED` 审核不通过

### 2.2 优先级 `priority`

- `P0`
- `P1`
- `P2`

### 2.3 可用动作 `availableActions`

- `CHANGE_STATUS`：状态流转
- `EDIT`：审核不通过后的编辑重提

### 2.4 需求ID `requirementId` 规则

- 新增需求按 `yyyyMMdd-序号` 生成，例如：`20260427-01`
- 同一天内序号递增，最少2位展示（`01`、`02` ... `99`、`100`）
- 旧历史数据保留原有值，不做回填

## 3. 统一返回

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

## 4. 接口明细

## 4.1 分页列表

- 方法：`GET`
- 路径：`/api/requirement/list`
- 权限：`requirement-manage:view`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `title` | string | 否 | 标题模糊查询 |
| `statusList` | string | 否 | 逗号分隔状态列表 |
| `priority` | string | 否 | `P0/P1/P2` |
| `proposerName` | string | 否 | 提出人模糊查询 |
| `businessLineCode` | string | 否 | 业务线编码 |
| `department` | string | 否 | 所属部门模糊查询 |
| `sortField` | string | 否 | `priority/createTime/updateTime` |
| `sortOrder` | string | 否 | `asc/desc` |
| `pageNum` | int | 否 | 默认 1 |
| `pageSize` | int | 否 | 默认 10 |

返回 `data`：`Page<TrackRequirement>`。

## 4.2 详情

- 方法：`GET`
- 路径：`/api/requirement/detail`
- 权限：`requirement-manage:view`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `requirementId` | string | 是 | 需求业务ID |

返回核心字段：

- `requirementId`
- `title`
- `status`
- `priority`
- `businessLineCode` / `businessLineName`
- `devTeamCode` / `devTeamName` / `devTeamDeptId`
- `expectedOnlineDate`
- `description`
- `screenshotFileId`（需求截图文件ID，可空）
- `proposerId` / `proposerName`
- `department`
- `createTime` / `updateTime`
- `availableActions[]`
- `logs[]`

## 4.3 新增需求

- 方法：`POST`
- 路径：`/api/requirement/add`
- 权限：`requirement-manage:add`

请求体字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `title` | string | 是 | 最大200字符 |
| `businessLineCode` | string | 是 | 业务线编码 |
| `priority` | string | 是 | `P0/P1/P2` |
| `expectedOnlineDate` | string | 是 | `YYYY-MM-DD` |
| `devTeamCode` | string | 是 | 负责开发团队编码（`SYS_DEPT.item_code`） |
| `description` | string | 否 | 需求描述 |
| `screenshotFileId` | string | 否 | 需求截图文件ID |

后端校验：

- `screenshotFileId` 非空时，必须存在于 `track_file_asset.file_id`。

## 4.4 状态变更

- 方法：`POST`
- 路径：`/api/requirement/status-change`
- 权限：`requirement-manage:status`

请求体字段：`requirementId`、`targetStatus`、`opinion(可选)`。

## 4.5 审核不通过后的编辑重提

- 方法：`POST`
- 路径：`/api/requirement/resubmit`
- 权限：`requirement-manage:resubmit`

请求体字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `requirementId` | string | 是 | 需求业务ID |
| `title` | string | 是 | 最大200字符 |
| `businessLineCode` | string | 是 | 业务线编码 |
| `priority` | string | 是 | `P0/P1/P2` |
| `expectedOnlineDate` | string | 是 | `YYYY-MM-DD` |
| `devTeamCode` | string | 是 | 负责开发团队编码（`SYS_DEPT.item_code`） |
| `description` | string | 否 | 需求描述 |
| `screenshotFileId` | string | 否 | 需求截图文件ID |

规则补充：

- 仅 `REJECTED` 状态可重提。
- 仅管理员或该需求提出人可重提。
- 至少有一个字段发生变化（含 `screenshotFileId` 变化）。

## 4.6 仪表盘统计

- 方法：`GET`
- 路径：`/api/requirement/dashboard-statistics`
- 权限：`requirement-manage:view`

返回 `data` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `weekSubmitCount` | number | 本周提交需求数（按 `createTime`） |
| `waitDevelopCount` | number | 待开发需求数（状态=`PENDING_REVIEW/SCHEDULING`） |
| `monthOnlineCount` | number | 本月已上线需求数（按日志 `toStatus=ONLINE` 去重统计） |
| `alertCount` | number | 异常进度告警数（排除 `REJECTED/OFFLINE`，且 `expectedOnlineDate<=今天+3天` 且状态非 `ONLINE`） |

## 4.7 仪表盘趋势

- 方法：`GET`
- 路径：`/api/requirement/dashboard-trend`
- 权限：`requirement-manage:view`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `days` | int | 否 | 仅支持 `7/30`，默认 `7` |

返回 `data`：数组，每日一条：

| 字段 | 类型 | 说明 |
|---|---|---|
| `date` | string | 日期（`YYYY-MM-DD`） |
| `newCount` | number | 当日新增需求数（按 `createTime`） |
| `onlineCount` | number | 当日上线需求数（按日志 `toStatus=ONLINE`） |

## 5. 依赖接口（图片）

- 上传图片：`POST /api/file/upload-image`（multipart/form-data）
- 预览图片：`GET /api/file/preview/{fileId}`（鉴权后返回二进制）

## 6. 2026-04-27 变更补充

### 6.1 负责开发团队来源调整

- 负责开发团队由独立“开发团队参数”切换为 `SYS_DEPT` 参数项。
- 可选条件：`status=0 && extraAttr='开发'`。
- 新增持久化字段：`devTeamDeptId`（`track_requirement.dev_team_dept_id`）。

### 6.2 状态变更权限收敛

- `IT/开发` 变更状态必须同时满足：
  - 当前用户包含 `developer` 角色；
  - 当前用户主部门ID等于需求 `devTeamDeptId`。
- `admin` 保持全状态流转能力。
