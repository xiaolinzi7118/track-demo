# 需求管理接口字段对照文档

## 1. 文档范围

- 模块：需求管理
- 基础路径：`/api/requirement`
- 对应后端实现：
  - `RequirementController`
  - `RequirementService`
- 对应数据库：
  - `track_requirement`
  - `track_log`（`log_type=requirement_manage`）

---

## 2. 状态/枚举定义

### 2.1 需求状态 `status`

- `PENDING_REVIEW`：待审核
- `SCHEDULING`：排期中
- `DEVELOPING`：开发中
- `TESTING`：测试中
- `ONLINE`：已上线
- `OFFLINE`：已下线
- `REJECTED`：审核不通过

### 2.2 优先级 `priority`

- `P0`
- `P1`
- `P2`

### 2.3 操作日志类型

- `log_type`：固定 `requirement_manage`
- `action_type`：
  - `CREATE`
  - `STATUS_CHANGE`
  - `EDIT_RESUBMIT`

### 2.4 可用操作 `availableActions`

- `actionType`：
  - `CHANGE_STATUS`
  - `EDIT`
- 当 `actionType=CHANGE_STATUS` 时：
  - `targetStatus`
  - `targetStatusName`
  - `label`
  - `needOpinion`

---

## 3. 接口清单

### 3.1 分页列表

- 方法：`GET`
- 路径：`/api/requirement/list`
- 权限：`requirement-manage:view`

#### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `title` | string | 否 | 需求标题，模糊匹配 |
| `statusList` | string | 否 | 状态列表，逗号分隔，如 `PENDING_REVIEW,REJECTED` |
| `priority` | string | 否 | `P0/P1/P2` |
| `proposerName` | string | 否 | 提出人，模糊匹配 |
| `businessLineCode` | string | 否 | 业务线编码 |
| `department` | string | 否 | 所属部门，模糊匹配 |
| `sortField` | string | 否 | `priority/createTime/updateTime`，默认 `createTime` |
| `sortOrder` | string | 否 | `asc/desc`，默认 `desc` |
| `pageNum` | int | 否 | 页码，默认 `1` |
| `pageSize` | int | 否 | 每页条数，默认 `10` |

#### 响应 `data`

- `Page<TrackRequirement>`，核心字段：
  - `content[]`
    - `requirementId`
    - `title`
    - `status`
    - `priority`
    - `proposerName`
    - `businessLineCode`
    - `businessLineName`
    - `department`
    - `createTime`
    - `updateTime`
    - `availableActions[]`
  - `totalElements`
  - `totalPages`
  - `size`
  - `number`

### 3.3 新增需求

- 方法：`POST`
- 路径：`/api/requirement/add`
- 权限：`requirement-manage:add`

#### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `title` | string | 是 | 最大长度 200 |
| `businessLineCode` | string | 是 | 业务线编码（字典） |
| `priority` | string | 是 | `P0/P1/P2` |
| `expectedOnlineDate` | string | 是 | 格式 `YYYY-MM-DD` |
| `devTeamCode` | string | 是 | 开发团队编码（字典） |
| `description` | string | 否 | 纯文本描述 |

#### 后端补齐字段

- `requirementId`（uuid v4）
- `status=PENDING_REVIEW`
- `proposerId/proposerName`
- `department`
- `createTime/updateTime`（服务器时间）
- 写入日志：`CREATE`

---

### 3.4 状态变更

- 方法：`POST`
- 路径：`/api/requirement/status-change`
- 权限：`requirement-manage:status`

#### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `requirementId` | string | 是 | 需求业务ID |
| `targetStatus` | string | 是 | 目标状态 |
| `opinion` | string | 否 | 操作意见；变更到 `REJECTED` 时必填 |

#### 规则

- 按后端状态机+角色强校验。
- 仅当 `availableActions` 中存在对应变更动作时允许提交。
- 变更到 `REJECTED` 且无原因时返回失败。
- 成功后更新 `status/updateTime` 并写日志：`STATUS_CHANGE`。

---

### 3.5 审核不通过后重提

- 方法：`POST`
- 路径：`/api/requirement/resubmit`
- 权限：`requirement-manage:resubmit`

#### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `requirementId` | string | 是 | 需求业务ID |
| `title` | string | 是 | 最大长度 200 |
| `businessLineCode` | string | 是 | 业务线编码 |
| `priority` | string | 是 | `P0/P1/P2` |
| `expectedOnlineDate` | string | 是 | `YYYY-MM-DD` |
| `devTeamCode` | string | 是 | 开发团队编码 |
| `description` | string | 否 | 纯文本描述 |

#### 规则

- 仅 `status=REJECTED` 可重提。
- 仅管理员或该需求提出人可重提。
- 必须至少一个字段发生实质变化。
- 成功后：
  - `status -> PENDING_REVIEW`
  - 更新 `updateTime`
  - 写日志：`EDIT_RESUBMIT`

## 4. 字典接口依赖

- 路径：`POST /api/dict-param/ids-list`
- 入参：
```json
{
  "paramIds": ["DICT2026042200000001", "DICT2026042200000002"]
}
```
- 约束：
  - 前端使用 `status===0` 的字典项
  - `businessLineCode/devTeamCode` 由后端二次校验有效性

---

## 5. 标准返回结构

所有接口统一返回：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- `code != 200` 代表业务失败（如参数错误、越权、状态流转非法）。
