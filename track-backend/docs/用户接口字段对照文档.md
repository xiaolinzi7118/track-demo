# 用户接口字段对照文档

## 1. 文档范围

- 功能：系统管理-用户管理
- 基础路径：`/api/user`
- 关联接口：
  - 用户管理主接口：`/api/user/*`
  - 部门下拉依赖：`/api/dict-param/dept-options`

---

## 2. 统一返回结构

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

- `code=200`：成功
- `code!=200`：业务失败（参数错误、越权、对象不存在等）

---

## 3. 权限要求

| 接口 | 权限 |
|---|---|
| `GET /api/user/list` | `system-user:view` |
| `GET /api/user/detail` | `system-user:view` |
| `POST /api/user/add` | `system-user:add` |
| `POST /api/user/update` | `system-user:edit` |
| `POST /api/user/delete` | `system-user:delete` |
| `POST /api/user/update-password` | `system-user:edit` |
| `POST /api/user/update-status` | `system-user:edit` |
| `GET /api/dict-param/dept-options` | `system-user:view` |

---

## 4. 接口明细

## 4.1 用户列表

- 方法：`GET`
- 路径：`/api/user/list`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `keyword` | string | 否 | 用户名/昵称模糊搜索 |

### 响应 `data`（数组）

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 用户ID |
| `username` | string | 用户名 |
| `nickname` | string | 昵称 |
| `avatar` | string \| null | 头像 |
| `status` | number | 状态：`1=启用`，`0=禁用` |
| `createTime` | datetime | 创建时间 |
| `isSuperAdmin` | boolean | 是否内置超级管理员 |
| `primaryDeptId` | number \| null | 主部门ID |
| `primaryDeptName` | string \| null | 主部门名称 |
| `roleIds` | number[] | 角色ID列表 |
| `roleNames` | string[] | 角色名称列表 |
| `roleId` | number \| null | 兼容字段：首个角色ID |
| `dataDeptIds` | number[] | 数据权限部门ID列表 |
| `dataDeptNames` | string[] | 数据权限部门名称列表 |

说明：内置超级管理员账号不会出现在列表中。

---

## 4.2 用户详情

- 方法：`GET`
- 路径：`/api/user/detail`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 用户ID |

### 响应 `data`

- 字段与“用户列表单行结构”一致。

---

## 4.3 新增用户

- 方法：`POST`
- 路径：`/api/user/add`

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `username` | string | 是 | 用户名，需唯一 |
| `password` | string | 是 | 密码 |
| `nickname` | string | 否 | 昵称 |
| `primaryDeptId` | number | 否 | 主部门ID，不传时后端尝试默认部门 |
| `status` | number | 否 | 默认 `1` |
| `roleIds` | number[] | 否 | 用户角色ID列表 |
| `roleId` | number | 否 | 兼容字段，未传 `roleIds` 时生效 |
| `dataDeptIds` | number[] | 否 | 数据权限部门ID列表 |

### 响应 `data`

- `User` 实体（基础字段）：
  - `id/username/nickname/avatar/primaryDeptId/status/isBuiltinSuperAdmin/createTime/updateTime`

---

## 4.4 编辑用户

- 方法：`POST`
- 路径：`/api/user/update`

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 用户ID |
| `nickname` | string | 否 | 昵称 |
| `primaryDeptId` | number | 否 | 主部门ID |
| `status` | number | 否 | 状态 |
| `roleIds` | number[] | 否 | 角色ID列表（传则全量覆盖） |
| `roleId` | number | 否 | 兼容单角色字段 |
| `dataDeptIds` | number[] | 否 | 数据权限部门ID列表（传则全量覆盖） |

### 响应 `data`

- `User` 实体（同新增响应结构）。

---

---

## 4.8 部门下拉（用户管理依赖）

- 方法：`GET`
- 路径：`/api/dict-param/dept-options`

### 响应 `data`（数组）

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 参数项ID（部门ID） |
| `paramId` | string | 参数ID，固定 `SYS_DEPT` |
| `itemCode` | string | 部门编码 |
| `itemName` | string | 部门名称 |
| `status` | number | 状态（仅返回 `0`） |
| `createBy` | string | 创建人 |
| `createTime` | datetime | 创建时间 |
| `updateBy` | string | 更新人 |
| `updateTime` | datetime | 更新时间 |

---

## 5. 业务规则说明

1. 内置超级管理员不可编辑、删改状态、重置密码。  
2. 用户主部门会被自动纳入其数据权限部门集合。  
3. 角色关系与数据部门关系在更新接口中为“覆盖式写入”。  
4. 用户ID为雪花ID（`BIGINT`），不再使用数据库自增。
