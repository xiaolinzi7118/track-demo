# 属性管理接口字段对照文档

## 1. 文档范围

- 功能：埋点管理-属性管理
- 基础路径：`/api/attribute`

---

## 2. 统一返回结构

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

---

## 3. 权限要求

| 接口 | 权限 |
|---|---|
| `GET /api/attribute/list` | `attribute:view` |
| `GET /api/attribute/detail` | `attribute:view` |
| `GET /api/attribute/all` | `attribute:view` |
| `POST /api/attribute/add` | `attribute:add` |
| `POST /api/attribute/update` | `attribute:edit` |
| `POST /api/attribute/delete` | `attribute:delete` |

---

## 4. 属性数据结构（TrackAttribute）

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 主键 |
| `attributeId` | string | 业务ID，如 `ATTR2026042700000001` |
| `attributeName` | string | 属性名称 |
| `attributeField` | string | 上报字段key |
| `attributeType` | string | `user/system/custom` |
| `sourceType` | string \| null | custom来源类型 |
| `sourceValue` | string \| null | 变量路径或静态值 |
| `interfaceId` | number \| null | `api_data` 来源时有效 |
| `interfacePath` | string \| null | 接口路径（后端填充） |
| `defaultValue` | string \| null | 默认值 |
| `status` | number | `0=生效`，`1=已删除` |
| `createBy` | string | 创建人 |
| `updateBy` | string | 更新人 |
| `createTime` | datetime | 创建时间 |
| `updateTime` | datetime | 更新时间 |

---

## 5. 接口明细

## 5.1 属性列表

- 方法：`GET`
- 路径：`/api/attribute/list`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `keyword` | string | 否 | 按 `attributeName/attributeField` 模糊搜索 |
| `attributeType` | string | 否 | `user/system/custom` |
| `pageNum` | number | 否 | 默认 `1` |
| `pageSize` | number | 否 | 默认 `10` |

响应 `data`：分页结构，内容为 `TrackAttribute[]`（仅 `status=0`）。

---

## 5.2 属性详情

- 方法：`GET`
- 路径：`/api/attribute/detail`

请求参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 主键ID |

---

## 5.3 新增属性

- 方法：`POST`
- 路径：`/api/attribute/add`

请求体：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `attributeName` | string | 是 | 属性名称 |
| `attributeField` | string | 是 | 属性字段 |
| `attributeType` | string | 是 | `user/system/custom` |
| `sourceType` | string | 条件必填 | `attributeType=custom` 时必填 |
| `sourceValue` | string | 条件必填 | `sourceType!=node_content` 时必填 |
| `interfaceId` | number | 条件必填 | `sourceType=api_data` 时必填 |
| `defaultValue` | string | 否 | 默认值 |

响应 `data`：新增后的 `TrackAttribute`。

---

## 5.4 更新属性

- 方法：`POST`
- 路径：`/api/attribute/update`

请求体：同新增属性，另需 `id`。

---

## 5.5 删除属性

- 方法：`POST`
- 路径：`/api/attribute/delete`

请求体：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 主键ID |

响应 `data`：`null`

说明：软删除（`status=1`）；若已被事件引用，返回错误 `"该属性已被事件引用，无法删除"`。

---

## 5.6 全量属性列表

- 方法：`GET`
- 路径：`/api/attribute/all`

说明：返回 `status=0` 的全量属性，供事件管理选择器使用。

---

## 6. 业务规则说明

1. 同一 `attributeType` 下，`attributeName` 在生效数据内唯一。  
2. `attributeType=custom` 时，来源类型支持：`node_content/api_data/global_object/local_cache/static_value`。  
3. `sourceType=api_data` 时必须传 `interfaceId`，后端自动填充 `interfacePath`。  
4. 属性业务ID生成规则：`ATTR + yyyyMMdd + 8位序号`。
