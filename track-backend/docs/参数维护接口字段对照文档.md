# 参数维护接口字段对照文档

## 1. 文档范围

- 功能：系统管理-参数维护（字典）
- 基础路径：`/api/dict-param`

---

## 2. 统一返回结构

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

---

## 3. 权限要求

| 接口 | 权限 |
|---|---|
| `GET /api/dict-param/list` | 角色硬校验：`developer` |
| `GET /api/dict-param/detail` | 角色硬校验：`developer` |
| `POST /api/dict-param/add` | 角色硬校验：`developer` |
| `POST /api/dict-param/update` | 角色硬校验：`developer` |
| `POST /api/dict-param/delete` | 角色硬校验：`developer` |
| `POST /api/dict-param/ids-list` | 无权限校验（登录后可用） |
| `GET /api/dict-param/dept-options` | `system-user:view` |

说明：内置超级管理员可通过超级管理员能力放行。

---

## 4. 数据结构

## 4.1 DictParam

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 主键ID |
| `paramId` | string | 业务参数ID（后端生成，如 `DICT20260424...`） |
| `paramName` | string | 参数名称 |
| `isSystem` | number | `0=普通`，`1=系统内置` |
| `status` | number | `0=生效`，`1=已删除` |
| `createBy` | string | 创建人 |
| `createTime` | datetime | 创建时间 |
| `updateBy` | string | 更新人 |
| `updateTime` | datetime | 更新时间 |
| `items` | DictParamItem[] | 参数项集合 |
| `deletedItemIds` | number[] | 待删除参数项ID集合（更新时使用） |

## 4.2 DictParamItem

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | number | 参数项ID |
| `paramId` | string | 所属参数ID |
| `itemCode` | string | 参数项编码 |
| `itemName` | string | 参数项名称 |
| `extraAttr` | string \| null | 附加属性 |
| `status` | number | `0=生效`，`1=已删除` |
| `createBy` | string | 创建人 |
| `createTime` | datetime | 创建时间 |
| `updateBy` | string | 更新人 |
| `updateTime` | datetime | 更新时间 |

---

## 5. 接口明细

## 5.1 参数列表

- 方法：`GET`
- 路径：`/api/dict-param/list`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `keyword` | string | 否 | 按 `paramName` 模糊搜索 |

### 响应 `data`

- `DictParam[]`（仅主表字段，不含 `items` 详情）。

---

## 5.2 参数详情

- 方法：`GET`
- 路径：`/api/dict-param/detail`

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 参数主键ID |

### 响应 `data`

- `DictParam` 对象，包含：
  - 主表字段
  - `items`（仅 `status=0` 的参数项）

---

## 5.3 新增参数

- 方法：`POST`
- 路径：`/api/dict-param/add`

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `paramName` | string | 是 | 参数名称（生效数据内唯一） |
| `items` | array | 否 | 参数项列表 |

`items[]` 元素结构：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `itemCode` | string | 是 | 参数项编码 |
| `itemName` | string | 是 | 参数项名称 |
| `extraAttr` | string | 否 | 附加属性 |

### 响应 `data`

- 新增后的 `DictParam`（含 `paramId`、`items`）。

说明：
- `paramId` 后端自动生成；
- `isSystem/status/createBy/updateBy/createTime/updateTime` 后端维护。

---

## 5.4 更新参数

- 方法：`POST`
- 路径：`/api/dict-param/update`

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 参数主键ID |
| `paramName` | string | 是 | 参数名称 |
| `items` | array | 否 | 参数项列表（新增/编辑） |
| `deletedItemIds` | number[] | 否 | 需要软删除的参数项ID |

`items[]` 元素结构：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 否 | 传则更新现有项，不传则新增 |
| `itemCode` | string | 是 | 参数项编码 |
| `itemName` | string | 是 | 参数项名称 |
| `extraAttr` | string | 否 | 附加属性 |

### 响应 `data`

- 更新后的 `DictParam`（含最新 `items`）。

---

## 5.5 删除参数

- 方法：`POST`
- 路径：`/api/dict-param/delete`

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `id` | number | 是 | 参数主键ID |

### 响应 `data`

- `null`

说明：删除为软删除，主表与参数项均置 `status=1`。

---

## 5.6 批量按参数ID查询（业务接口依赖）

- 方法：`POST`
- 路径：`/api/dict-param/ids-list`

### 请求体

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `paramIds` | string[] | 是 | 参数ID列表（如 `DICT2026042200000001`） |

### 响应 `data`

- `DictParam[]`
  - 每项包含 `items`
  - 返回顺序按请求 `paramIds` 顺序组织
  - `items` 不做状态过滤（前端按业务自行过滤，如 `status===0`）

---

## 5.7 部门下拉

- 方法：`GET`
- 路径：`/api/dict-param/dept-options`

### 响应 `data`

- `DictParamItem[]`
  - 固定来源：`paramId=SYS_DEPT`
  - 仅返回 `status=0`

---

## 6. 业务规则说明

1. 参数名称在生效数据内唯一。  
2. 系统参数（`isSystem=1`）不可重命名、不可删除。  
3. 参数项校验：
   - `itemCode/itemName` 必填
   - 同一请求内 `itemCode` 不可重复
   - 更新时 `id` 必须属于当前参数。  
4. 参数ID生成规则：`DICT + yyyyMMdd + 8位序号`。
