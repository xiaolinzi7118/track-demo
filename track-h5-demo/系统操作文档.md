# 埋点管理平台与H5上报系统 操作文档

## 一、系统概述

### 1.1 系统架构

本系统采用**前后端分离+四层架构**设计，包含四大核心模块：
- **H5埋点上报SDK**（track-sdk）：轻量级JavaScript SDK，负责采集用户行为数据并上报
- **H5演示应用**（track-h5-demo）：集成SDK的示例电商H5应用
- **埋点管理后台**（track-admin）：Vue3 + Element Plus 管理平台，用于埋点配置和数据分析
- **后端服务**（track-backend）：Spring Boot + H2 后端服务，提供RESTful API和数据存储

## 三、埋点SDK完整API文档

### 3.1 SDK引入方式

```html
<!-- 方式1：直接引入（推荐生产环境） -->
<script src="/path/to/track-sdk.min.js"></script>

<!-- 方式2：ES Module导入（开发阶段） -->
import TrackSDK from 'track-sdk'
```

### 3.2 初始化配置

```javascript
// 完整配置示例
TrackSDK.init({
  // 必填：上报服务器地址
  serverUrl: 'http://localhost:8080',
  // 必填：应用唯一标识
  appId: 'demo-app',
  // 可选：是否开启调试模式（默认：false）
  debug: true,
  // 可选：自动埋点配置
  autoTrack: {
    // 自动采集页面曝光（默认：true）
    pageView: true,
    // 自动采集点击事件（默认：true）
    click: true
  },
  // 可选：上报配置
  maxQueueSize: 20,        // 批量上报阈值（默认：20条）
  flushInterval: 5000      // 定时上报间隔（默认：5000ms）
})
```

### 3.3 核心API方法

#### 3.3.1 初始化（init）

```javascript
/**
 * 初始化SDK
 * @param {Object} config - 配置对象
 * @returns {Object} SDK实例
 */
TrackSDK.init(config)
```

#### 3.3.2 手动埋点上报（track）

```javascript
/**
 * 上报自定义事件
 * @param {string} eventCode - 事件编码（与管理平台配置对应）
 * @param {string} eventType - 事件类型（'page_view' | 'click' | 'custom'）
 * @param {Object} params - 自定义参数
 */
TrackSDK.track('event_code', 'click', {
  productId: '12345',
  productName: '智能手机'
})
```

#### 3.3.3 页面曝光快捷方法（trackPageView）

```javascript
/**
 * 上报页面曝光事件
 * @param {string} eventCode - 事件编码
 * @param {Object} params - 自定义参数
 */
TrackSDK.trackPageView('home_page_view', {
  pageTitle: '首页',
  channel: 'wechat'
})
```

#### 3.3.4 点击事件快捷方法（trackClick）

```javascript
/**
 * 上报点击事件
 * @param {string} eventCode - 事件编码
 * @param {Object} params - 自定义参数
 */
TrackSDK.trackClick('add_cart', {
  productId: '12345',
  quantity: 1
})
```

#### 3.3.5 用户识别（setUserId/setUserInfo）

```javascript
/**
 * 设置用户ID（用户登录后调用）
 * @param {string} userId - 用户唯一标识
 */
TrackSDK.setUserId('user_10086')

/**
 * 设置用户信息
 * @param {Object} userInfo - 用户信息对象
 */
TrackSDK.setUserInfo({
  userId: 'user_10086',
  nickname: '张三',
  vipLevel: 'gold'
})
```

#### 3.3.6 获取会话ID（getSessionId）

```javascript
/**
 * 获取当前会话ID
 * @returns {string} sessionId
 */
const sessionId = TrackSDK.getSessionId()
console.log('当前会话ID:', sessionId)
```

### 3.4 自动采集机制

#### 3.4.1 页面曝光自动采集

**触发条件**：
- 页面首次加载完成
- 路由切换（SPA应用）
- 页面从隐藏状态变为可见（visibilitychange）

**采集的数据**：
- 页面URL
- 页面标题
- 进入时间
- 离开时间（用于计算停留时长）

#### 3.4.2 点击事件自动采集

**触发条件**：
- 点击带有 `data-track-id` 或 `track-id` 属性的元素
- 支持父级元素向上查找（最多查找到body）

**标记方式**：
```html
<!-- 基本用法 -->
<button data-track-id="submit_order">提交订单</button>

<!-- 父元素标记（子元素点击也会触发） -->
<div data-track-id="product_card">
  <img src="product.jpg" />
  <p>商品名称</p>
  <span>¥999</span>
</div>

<!-- Vue动态绑定 -->
<button :data-track-id="'btn_' + type">按钮</button>
```

**自动采集的参数**：
- `elementTag`：元素标签名（如：BUTTON、DIV）
- `elementText`：元素文本内容（前50个字符）
- `elementClass`：元素class属性
- `elementId`：元素id属性

### 3.5 上报策略

#### 3.5.1 批量上报

- 当队列中事件数达到 `maxQueueSize`（默认20条）时触发上报
- 减少HTTP请求次数，减轻服务器压力

#### 3.5.2 定时上报

- 每 `flushInterval`（默认5000ms）检查一次队列
- 队列非空则立即上报

#### 3.5.3 页面卸载上报

- 监听 `beforeunload` 事件
- 页面卸载前强制上报剩余数据
- 使用 `navigator.sendBeacon` 保证上报成功率

#### 3.5.4 失败重试

- 上报失败的事件会放回队列
- 下次上报时重新尝试（最多3次）

## 五、埋点管理平台操作指南

### 5.2 首页仪表盘（Dashboard）

登录成功后进入首页，展示核心统计指标：
- **埋点配置总数**：系统中已配置的埋点数量

- **点击交互埋点**：点击类型的埋点数量

### 5.3 埋点配置管理

#### 5.3.1 配置列表

进入「埋点配置」页面，可以查看所有埋点配置：
- 支持按**事件类型**筛选（页面曝光/点击交互）
- 支持按**关键词**搜索（事件名称/事件编码）
- 列表展示：事件名称、事件编码、事件类型、描述、生效页面、状态、创建时间

#### 5.3.2 新增埋点配置

1. 点击「新增配置」按钮
2. 填写配置表单：
   | 字段 | 说明 | 示例 |
   |------|------|------|
   | 事件名称 | 埋点的业务名称 | 首页分享按钮点击 |
   | 事件编码 | 埋点唯一标识（与data-track-id对应） | home_btn_share |
   | 事件类型 | 页面曝光/点击交互 | 点击交互 |
   | 状态 | 启用/禁用 | 启用 |
   | 描述 | 详细描述该埋点的业务含义 | 用户点击首页右上角分享按钮 |
   | 生效页面URL正则 | 只有匹配的页面才会触发该埋点 | ^.*/#/$ |
   | 参数配置 | 可扩展的参数定义 | - |

3. 点击「确定」保存

#### 5.3.3 编辑/删除配置

- **编辑**：点击操作列的「编辑」按钮，修改配置信息
- **删除**：点击操作列的「删除」按钮，删除该埋点配置

### 5.4 数据回检查询

进入「埋点数据」页面，可查询SDK上报的原始数据：

#### 5.4.1 查询条件

- **事件编码**：按埋点编码精确查询
- **事件类型**：按类型筛选（页面曝光/点击交互）
- **用户ID**：按用户ID查询

#### 5.4.2 数据列表

展示字段：
- 事件编码
- 事件类型（标签展示）
- 页面URL（超长截断，hover显示完整）
- 用户ID
- 会话ID
- 停留时长（仅页面曝光事件显示）
- 参数（JSON格式化展示）
- 事件时间
- IP地址

#### 5.4.3 数据统计卡片

页面底部展示统计概览：
- 上报数据总数
- 页面曝光事件数
- 点击交互事件数
- 总曝光时长（秒）

2. 按事件编码或时间筛选
3. 确认数据已正确上报并展示

## 七、常见问题（FAQ）

### Q1: SDK初始化失败，提示 `serverUrl is required`？

**A:** 检查初始化代码，确保 `serverUrl` 参数已正确配置：
```javascript
TrackSDK.init({
  serverUrl: 'http://localhost:8080',  // 确保此项正确
  appId: 'demo-app',
  debug: true
})
```

### Q2: 埋点数据没有上报？

**排查步骤**：
1. 检查 `serverUrl` 是否正确，后端服务是否已启动
2. 检查浏览器控制台是否有报错信息
3. 检查 Network 面板是否有上报请求（路径：`/api/track-data/batch-report`）
4. 确认埋点元素是否正确添加了 `data-track-id` 属性
5. 检查管理后台该埋点配置的状态是否为「启用」
6. 检查该埋点配置的 `urlPattern` 是否匹配当前页面URL

### Q3: 如何查看上报的埋点数据？

**A:** 两种方式：
1. **SDK日志**：开启 `debug: true`，在浏览器控制台查看
2. **管理后台**：登录管理后台 → 进入「埋点数据」页面 → 按条件查询

### Q4: SPA应用路由切换时页面曝光事件不触发？

**A:** SDK已支持 `popstate` 事件监听。如果使用Vue Router，确保使用了 `createWebHashHistory` 或 `createWebHistory`，SDK会自动检测路由变化。

### Q5: 页面停留时长是如何计算的？

**A:** 停留时长计算逻辑：
1. 页面曝光时记录 `pageEnterTime`
2. 监听 `visibilitychange` 事件，页面隐藏时计算停留时间
3. 监听 `beforeunload` 事件，页面卸载时上报最终停留时间
4. 时长单位：毫秒（ms）

### Q6: 上报失败的数据会丢失吗？

**A:** SDK有重试机制：
- 上报失败的事件会放回队列
- 下次上报时会重新尝试
- 最多重试3次
- 队列满时旧数据会被新数据覆盖（默认20条）

### Q7: 支持跨域上报吗？

**A:** 支持。后端已配置CORS跨域（`CorsConfig.java`），允许所有来源的请求。生产环境建议配置具体的域名白名单。

## 八、性能与安全

### 8.1 性能影响

- **SDK体积**：压缩后约5KB，对页面加载影响极小
- **上报策略**：批量上报+定时上报，减少HTTP请求
- **无阻塞**：所有操作异步执行，不影响主线程
- **离线存储**：待实现（可使用IndexedDB存储离线数据）

### 8.2 安全考虑

- **数据加密**：建议生产环境使用HTTPS协议
- **敏感信息**：禁止在埋点参数中包含敏感信息（密码、手机号等）
- **跨站脚本**：SDK内部已做XSS防护，但业务方需确保参数安全
- **IP收集**：后端通过HTTP头获取客户端IP，不依赖前端上报
