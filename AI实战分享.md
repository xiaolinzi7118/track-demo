# AI Agent 驱动的埋点全链路系统 —— 从零到一的实战分享

## 一、前言

本文将分享如何利用 **Claude Code**（Anthropic 推出的 AI 编程助手）从零构建一套完整的前端埋点解决方案。整个项目涵盖**埋点配置管理平台、H5 埋点 SDK、数据回检**三大核心模块，全程由 AI Agent 主导开发，开发工程师负责需求输入与方向把控。

## 二、AI 辅助开发环境搭建

### 2.1 安装 Claude Code

Claude Code 是 Anthropic 公司推出的 AI 编程助手，支持 **CLI 命令行** 和 **VS Code 插件** 两种使用方式。

#### 方式一：VS Code 插件安装（推荐）

1. 打开 VS Code 扩展商店（`Ctrl+Shift+X`）
2. 搜索 **"Claude Code"** 并安装
3. 安装后侧边栏会出现 Claude Code 图标，点击即可使用

#### 方式二：CLI 命令行安装

```bash
# 通过 npm 全局安装
npm install -g @anthropic-ai/claude-code

# 在项目目录中启动
cd your-project
claude
```

### 2.2 使用 ccSwitch 管理 AI 模型切换

在实际开发中，我们通常需要在不同 AI 模型之间切换以平衡**能力**与**成本**。[ccSwitch](https://github.com/anthropics/claude-code) 是一款便捷的 Claude Code 模型切换工具：

- **Claude Opus** — 最强模型，适合复杂架构设计和核心逻辑开发
- **Claude Sonnet** — 均衡模型，适合日常功能开发和代码编写
- **Claude Haiku** — 轻量模型，适合简单修改和快速问答

> 本项目开发过程中主要使用 **Claude Sonnet** 模型完成日常编码，在架构设计等关键环节切换至 **Claude Opus** 获取更优的方案输出。

### 2.3 AI 辅助开发的基本流程

```
步骤1：描述需求
   ↓  用自然语言清晰描述你要什么
步骤2：AI 分析与规划
   ↓  AI 阅读项目代码，理解上下文，制定实施方案
步骤3：方案确认
   ↓  开发工程师审查 AI 的方案，确认或调整方向
步骤4：AI 执行实施
   ↓  AI 自动创建文件、编写代码、配置依赖
步骤5：验证与迭代
   ↓  运行项目，发现问题后让 AI 修复，循环迭代
```

### 2.4 AI 与开发工程师的协作关系

AI 模型不是简单的代码生成器，而是一个能理解上下文、自主决策、多步执行的编程 Agent。开发工程师从"逐行写代码的人"转变为"指导 AI 写代码的人"，专注于需求定义、技术选型和方案审查。

一套**完整的前端埋点解决方案**，覆盖从埋点配置、数据采集到数据验证的全链路。

### 3.3 技术栈一览

| 模块 | 技术选型 | 说明 |
|------|---------|------|
| **后端服务** | Spring Boot 2.7.18 + JPA + H2 | 轻量级 Java 后端，内嵌数据库零部署 |

## 四、分阶段实施过程

### 第一阶段：项目初始化与后端服务搭建

**目标：** 搭建后端 API 服务，定义数据模型，实现核心接口。

#### 对 AI 的 Prompt 示例：
> "帮我创建一个 Spring Boot 项目，用于埋点数据管理。需要支持埋点配置的增删改查、埋点数据上报和查询。使用 JPA + H2 数据库。"

#### AI 完成的工作：

1. **初始化 Spring Boot 项目** — 生成 pom.xml，配置 Maven 依赖
2. **设计 4 个核心实体类：**
   - `User` — 系统用户（用户名、密码、昵称、头像、状态）
   - `TrackConfig` — 埋点配置（事件名称、事件编码、事件类型、参数配置JSON、URL正则、启用状态）
   - `TrackData` — 埋点数据（事件编码、URL、参数JSON、用户ID、会话ID、UA、IP、停留时长）
   - `ApiInterface` — API 接口定义（接口名称、路径、描述）
3. **实现 Controller → Service → Repository 三层架构**
4. **配置 CORS 跨域** — 允许前端管理平台和 SDK 跨域访问
5. **配置 H2 内嵌数据库** — 零部署，数据文件持久化存储
6. **实现统一响应格式 Result** — `{code, message, data}`

#### 关键产出：
- 20+ 个 RESTful API 接口
- 完整的 CRUD 操作
- 自动初始化默认管理员账号（admin/123456）

### 第二阶段：管理平台前端开发

**目标：** 开发埋点配置管理后台，支持配置管理、数据查看和统计分析。

#### 对 AI 的 Prompt 示例：
> "帮我用 Vue 3 + Element Plus 创建一个埋点管理后台，包含登录页、仪表盘、埋点配置管理页面、数据回检页面。需要对接上一阶段的后端接口。"

#### AI 完成的工作：

1. **初始化 Vue 3 项目** — 配置 Vite、Vue Router、Pinia、Element Plus
2. **实现登录认证模块**
   - 登录页面（用户名/密码）
   - Token 状态管理（Pinia Store）
   - 路由守卫（未登录自动跳转）
3. **布局与导航**
   - 侧边栏菜单 + 顶部导航
   - 动态菜单渲染
   - 面包屑导航
4. **仪表盘页面**
   - 埋点配置统计（总数、页面曝光数、点击事件数）
   - 数据上报统计（总事件数、PV、点击数）
   - 趋势图表展示
5. **埋点配置管理页面**
   - 配置列表（分页、搜索、筛选）
   - 新增/编辑配置弹窗
   - 参数动态配置（支持多种参数来源类型）
   - URL 正则配置（控制埋点生效范围）
   - 启用/禁用开关
6. **API 接口管理页面**
   - 管理可供埋点参数引用的 API 接口

7. **数据回检页面**
   - 埋点数据列表（支持事件编码、类型、用户ID 筛选）
   - 数据详情查看（参数内容展开）
   - 统计概览（总数、PV、点击、时长）
8. **系统管理**
   - 用户管理
   - 数据重置功能

#### 关键产出：
- 完整的 SPA 管理后台
- 6 个功能页面
- 响应式布局，企业级 UI

### 第三阶段：H5 埋点 SDK 开发

**目标：** 开发一个轻量级的 H5 埋点 SDK，支持自动埋点和手动埋点，无框架依赖。

#### 对 AI 的 Prompt 示例：
> "帮我开发一个 H5 埋点 SDK，使用原生 JavaScript，支持自动追踪页面曝光和点击事件。需要从后端拉取配置，根据配置自动采集数据并上报。使用 Rollup 打包成 UMD 格式。"

#### AI 完成的工作：

1. **SDK 核心架构设计**
   - 初始化模块 — 配置加载与服务端同步
   - 事件追踪引擎 — 监听 DOM 事件，采集埋点数据
   - 配置管理器 — 解析远程配置，匹配当前页面 URL
   - 数据上报队列 — 批量上报机制，定时刷新

2. **自动追踪能力**
   - **页面曝光（page_view）** — 自动监测页面加载和路由切换
   - **点击事件（click）** — 全局点击事件监听与 DOM 元素匹配
   - **声明式埋点** — 通过 `data-track-id` 属性标记可追踪元素
   - **页面停留时长** — 自动记录用户在页面的停留时间

3. **动态参数采集（5 种参数来源）**
   - `node_content` — 从点击的 DOM 元素提取文本内容
   - `api_data` — 从接口响应缓存中提取数据
   - `global_object` — 从 window 全局对象读取
   - `local_cache` — 从 localStorage 读取
   - `static_value` — 配置中的静态值

4. **高级功能**
   - **网络请求拦截** — 拦截 XMLHttpRequest，缓存 API 响应数据
   - **会话管理** — UUID 持久化标识 + 会话 ID
   - **队列化批量上报** — 每 5 秒或队列满 20 条自动上报
   - **页面可见性监听** — 切换标签页时触发数据上报
   - **URL 正则匹配** — 支持正则表达式控制埋点生效页面

5. **构建与发布**
   - Rollup 打包配置
   - Babel 转译（兼容性）
   - Terser 代码压缩
   - UMD 格式输出（支持 `<script>` 标签引入和模块化使用）

#### SDK 使用示例：

```html
<!-- 引入 SDK -->
<script src="/track-sdk.min.js"></script>

<!-- 初始化 -->
<script>
TrackSDK.init({
  serverUrl: 'http://localhost:8080',
  appId: 'bank-app',
  debug: true,
  autoTrack: { pageView: true, click: true }
})
</script>

<!-- 声明式埋点 -->
<button data-track-id="transfer_btn">转账汇款</button>
```

#### 关键产出：
- 零依赖的埋点 SDK
- 5 种动态参数采集方式
- 批量上报 + 网络拦截
- UMD 格式，一行代码集成

## 八、总结与展望

本项目验证了 **AI Agent 可以作为主力开发者**，在开发工程师指导下完成从架构设计到功能实现的完整开发流程。关键成功因素：

1. **清晰的需求描述** — 越精确的 Prompt，产出质量越高
2. **分阶段推进** — 将大项目拆分为小任务，逐步交付
3. **持续验证** — 每个阶段完成后立即验证，快速迭代
4. **开发工程师把控方向** — 技术选型和架构决策由开发工程师主导

> 这套埋点系统从零到一，完全由 AI Agent 驱动完成。它证明了：AI 不是替代开发工程师，而是成为开发工程师最强大的协作伙伴。未来的开发模式，将是**开发工程师定义目标 + AI 高效执行**的全新范式。
